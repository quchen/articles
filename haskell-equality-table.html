Explanation of special cases

  • NaN is unequal to itself according to the floating point standard.
  • "" is identical to [] :: [Char], so they are equal.
  • [] cannot be compared to itself directly, because the concrete type of the contained values is unknown, and the right Eq instance cannot be found. Adding explicit type annotations solves this issue, ([] :: [Int]) == [] is True.
